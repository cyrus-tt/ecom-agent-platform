# AI-Native 企业级产品案例研究（2025-2026）

> 调研日期：2026-05-15
> 目的：了解美国市场 AI 替代员工的真实产品，提取可复用模式

---

## 市场背景

- PwC 2025.5 调查：79% 组织已在生产环境跑 AI Agent
- Agentic AI 市场：$5.25B（2024）→ $7.84B（2025）→ 预估 $52.62B（2030）
- Klarna 教训：全面用 AI 替代 700 客服后翻车，2025-2026 重新招人。行业共识从"替代所有人"转向"替代特定任务类别，保留人类判断"

## 标杆公司逐个拆解

### 1. Cognition (Devin) — 软件工程

| 维度 | 详情 |
|---|---|
| 替代谁 | 初级开发（写代码、调试、部署） |
| 自主度 | 高。Devin 3.0（2026）支持动态重规划，无需人类干预 |
| 架构 | 单 Agent + 持久化环境（shell/browser/editor）。Observe-Plan-Act-Replan 循环 |
| 商业模式 | 订阅制，从 $500/月降到 $20/月（Core），企业版定制 |
| 验证 | 高盛试点 12,000 开发者旁边跑 Devin，效率提升 20%。估值谈判 $25B（2026.4） |
| **启示** | 定价为商品（$20/月），定位为"团队成员"而非"工具"。企业粘性来自代码库定制微调 |

### 2. Sierra AI — 客户体验

| 维度 | 详情 |
|---|---|
| 替代谁 | L1 客服（聊天/语音/邮件/SMS/WhatsApp） |
| 自主度 | 高。Ramp 公司用 Sierra 实现 90% 自主解决率 |
| 架构 | Agent Operating System — 多渠道 Agent 平台。Ghostwriter（2026.3）用自然语言创建 Agent |
| 商业模式 | **按结果计费** — 每解决一个 case 收费，不按座位 |
| 验证 | $150M ARR（不到 2 年）。$15.8B 估值。服务 40% 的 Fortune 50 |
| **启示** | 按结果计费是商业模式创新。对齐了供应商和客户的激励 |

### 3. Harvey AI — 法律

| 维度 | 详情 |
|---|---|
| 替代谁 | 律所助理级任务（备忘录、尽调报告、文件审阅） |
| 自主度 | 中高。Agent 端到端执行但在决策点暂停让律师审核 |
| 架构 | 500+ 专业 Agent + Agent Builder 自定义工作流。多 Agent 流水线 |
| 商业模式 | 企业订阅。日处理 400K+ 查询 |
| 验证 | $11B 估值。10 万+ 律师使用。Am Law 100 大多数 + 500 企业法务 |
| **启示** | 在监管领域，Agent 替代的是产出量，不是判断力。"智能替代层级" — 初级律师的工作量交给 Agent，高级律师的判断变得更有价值 |

### 4. ServiceNow — IT/HR/财务运营

| 维度 | 详情 |
|---|---|
| 替代谁 | L1 IT 服务台、HR 处理、采购、安全分类 |
| 自主度 | L1 任务完全自主。IT Service Desk AI Specialist（2026 Q2）零人工解决 VPN/密码/软件安装 |
| 架构 | 平台原生 Specialist 嵌入 ServiceNow 工作流引擎。与 NVIDIA 合作优化推理 |
| 验证 | 内部部署 IT case 解决速度提升 99%。客户部署 91% 无需转人 |
| **启示** | 已有平台的公司有巨大分发优势——已拥有工作流，加 Agent 是升级不是替换 |

### 5. Salesforce Agentforce — CRM/销售/服务

| 维度 | 详情 |
|---|---|
| 替代谁 | 客服代表、SDR、IT 服务台 |
| 自主度 | 中高。自有帮助站 83% 自主解决率 |
| 架构 | 垂直行业 Agent 包（零售/金融/HR 等），编排器模式 + 护栏 |
| 验证 | 6,000+ 付费客户。1-800Accountant 报税周 90% case 自动处理 |
| **启示** | 垂直包装比通用平台好卖。"金融行业客服 Agent"比"通用 AI Agent"成交率高 |

### 6. Truewind — 会计/财务

| 维度 | 详情 |
|---|---|
| 替代谁 | 初级会计（分类、对账、总账、月结） |
| 自主度 | 中。Workpaper Agent 自动分类交易，会计从"执行者变成审核者" |
| **启示** | "执行者到审核者"是最诚实的定位。不是"替代团队"，是"5 人团队现在监督 50 人的产出" |

## 人类监督模式（三种主流）

| 模式 | 运作方式 | 适用场景 |
|---|---|---|
| **Human-in-the-loop (HITL)** | Agent 在定义好的检查点暂停，需要人类明确批准才继续 | 高风险：财务交易、法律文件、医疗 |
| **Human-on-the-loop (HOTL)** | Agent 在护栏内自主运行，人类监控看板处理异常 | 中风险：客服、IT 工单、数据处理 |
| **有界自主 (Bounded Autonomy)** | Agent 有"风险预算"——低风险自动执行，中风险记录，高风险要审批 | 最成熟的企业模式；符合 EU AI Act |

**行业最佳实践：风险分级自主权。** 每个 Agent 动作按风险分级：
- 低 = 自动执行
- 中 = 执行 + 记录 + 异步审核
- 高 = 暂停 + 请求审批

ServiceNow、Harvey、Sierra 都是这么做的。

## 生产级架构模式

| 模式 | 谁在用 | 优势 |
|---|---|---|
| **ReAct 循环** | Devin、单任务 Agent | 简单可调试，适合线性任务 |
| **编排器-工人** | Anthropic Claude Research, Harvey | 并行探索，比单 Agent 提升 90%+ |
| **Manager 模式** | ServiceNow, Salesforce | 可预测，企业友好，映射组织架构 |
| **DAG 工作流** | LangGraph 系统、企业流水线 | 生产最稳定，确定性路由 + Agent 节点 |
| **交接模式** | Sierra 多渠道路由 | 多领域问题，单 Agent 覆盖不了 |

**真相：没有任何生产系统只用一种模式。** 最好的系统是组合——DAG 走 happy path，ReAct 在单个节点内灵活处理，编排器兜底异常。

## 商业模式演变

| 模式 | 2025 占比 | 2026 占比 | 代表 |
|---|---|---|---|
| 按座位 | 21% | 15% | 传统 SaaS |
| 混合（基础 + 用量） | 27% | 41% | 大多数 AI 厂商 |
| 按结果（每解决一个 case） | 萌芽 | 标准（CX） | Sierra $按解决, Zendesk $1.50, HubSpot $0.50 |

## 对我们项目的 6 个可执行启示

1. **从有界自主开始，不是全自主。** Klarna 证明了"替代所有人"会失败。从 Day 1 把审批门控建进架构
2. **按结果计费是解锁点。** Agent 可验证地完成了任务才收费，消除"AI 到底有没有用"的疑问
3. **垂直包装好过横向平台。** "电商商品管理 AI Agent"比"通用 AI Agent"好卖
4. **"执行者到审核者"是诚实定位。** 不卖"替代团队"，卖"5 人监督 50 人产出"
5. **组合架构是必须的。** DAG + ReAct + 编排器，单一模式扛不住
6. **风险分级动作分类。** 每个 Agent 动作打风险标签，决定自动执行/记录/暂停审批
